# 응답 데이터의 JSON 구조 분석 및 각 데이터 필드 의미 작성

## 구조 분석

fetch("https://jsonplaceholder.typicode.com/todos/1")
-> 해당 URL 주소에 데이터 요청

.then((response) => response.json())
-> 서버가 응답을 하면 JSON 형식으로 변환

.then((json) => console.log(json));
-> 변환이 완료되면 JSON 값으로 받아와서 콘솔에 출력

## 데이터 필드 의미

{
"userId": 1, //사용자 고유 식별 번호
"id": 1, // 해당 항목 자체의 고유 식별 번호
"title": "delectus aut autem", // 제목
"completed": false // 완료 여부(현재는 false 이므로 완료되지 않음)
}

# Request 와 Response 차이

- Request -> 클라이언트가 서버에 데이터를 요청 (예: 메뉴판을 보고 음식 주문)
- Response -> 서버가 요청을 받으면 클라이언트에 데이터 제공 (예: 주문 받은 음식 제공)

# API 키가 필요한 이유 및 보안상 주의점

필요한 이유

- 누가 요청했는지 확인 용도
- 얼마나 사용했는지 기록하는 용도
- 사용 가능 권한 및 기능을 제한하는 용도

보안상 주의점

- 절대로 GitHub 에 공개 금지
- 비밀번호처럼 관리해야함
- 환경변수로 안전하게 보관하기
- 코드에 직접 작성 금지
